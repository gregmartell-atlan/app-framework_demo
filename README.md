# atlan-github-app

Native Atlan App Framework v3 connector for **GitHub** (cloud + GHE).
Catalogues repositories, issues, and pull requests as Atlan `ApiSpec` /
`ApiPath` assets.

## Why this layout

This repo is built to the **v3-readiness checklist** at
`atlanhq/application-sdk/docs/standards/v3-readiness.md`. Every layout
decision below maps to a checklist item:

| Checklist item | Implementation |
|----------------|----------------|
| §1 SDK pin `atlan-application-sdk==3.4.0` | `pyproject.toml` |
| §1 Single `App` subclass with `@task` only | `app/connector.py` extends `BaseMetadataExtractor` |
| §1 Typed `Input`/`Output` at every boundary | `app/contracts.py` (Pydantic + `MaxItems` + `FileReference`) |
| §1 `Handler` with typed signatures | `app/handler.py` uses `AuthInput/AuthOutput`, etc. |
| §1 `create_async_atlan_client` + typed credentials | `app/credentials.py` registers `GitHubTokenCredential` |
| §2 `contract/app.pkl` + generated artifacts committed | `contract/app.pkl` → `app/generated/` |
| §2 `poe generate` task wired | `pyproject.toml` `[tool.poe.tasks]` |
| §3 `FROM registry.atlan.com/public/app-runtime-base:3` | `Dockerfile` |
| §3 No `CMD` / `ENTRYPOINT` override | `Dockerfile` |
| §3 `ATLAN_APP_MODULE` set | `Dockerfile` |
| §4 App instantiation test | `tests/integration/test_boot_probe.py::test_app_instantiation` |
| §4 Per-task happy-path tests | `tests/integration/test_e2e_replay.py` |
| §4 Contract round-trip | `tests/unit/test_contracts.py` |
| §4 Credential registration | `tests/unit/test_credentials.py::test_credential_type_is_registered` |
| §4 Boot probe | `tests/integration/test_boot_probe.py::test_handler_service_serves_committed_manifest` |
| §4 Scenario coverage matrix | `tests/e2e/README.md` |
| §5 SDK-triggered checks via reusable workflow | `.github/workflows/v3-readiness.yaml` |

## Getting started (local dev on Mac)

See [`tests/e2e/RUN_LIVE.md`](tests/e2e/RUN_LIVE.md) for the one-paste setup
that runs the connector live against your `gregmartell-atlan` GitHub repos.

Quick version:

```bash
brew install pkl uv
uv sync
cd contract && pkl project resolve && cd ..
uv run poe generate                     # regenerate app/generated/* from contract/app.pkl
uv run poe check-v3                     # SDK static checker
uv run poe test                         # unit + integration (replayed)
GITHUB_TOKEN=$(gh auth token) uv run poe dev   # boots handler+worker on :8000
```

## Asset model

| GitHub entity | Atlan asset type | qualifiedName pattern |
|---------------|------------------|------------------------|
| Repository    | `ApiSpec`        | `{conn_qn}/{owner}/{repo}` |
| Issue         | `ApiPath`        | `{conn_qn}/{owner}/{repo}/issues/{number}` |
| Pull request  | `ApiPath`        | `{conn_qn}/{owner}/{repo}/pulls/{number}` |

Issues and pull requests are linked to their parent repo via
`api_specification = ApiSpec.ref_by_qualified_name(repo_qn)` — using
`uniqueAttributes.qualifiedName` (NOT `attributes.qualifiedName`) so the
publish step's Atlas calls succeed (V2 verification rule).

## Credential type

`GitHubTokenCredential` is registered with the SDK's `CredentialTypeRegistry`
at module load. Secret-store entries with `"type": "github_token"` resolve
into a typed credential whose `to_headers()` returns:

```
Authorization: Bearer <token>
Accept: application/vnd.github+json
X-GitHub-Api-Version: 2022-11-28
```
