# End-to-End Test Scenarios

This directory contains E2E tests for the GitHub App Framework v3 connector.

## Scenario Coverage Matrix

| # | Scenario | Test Type | Expected Result | Notes |
|---|----------|-----------|-----------------|-------|
| 1 | Auth — valid PAT | Live | Success, returns user login | Tests `handle_auth` with real GitHub token |
| 2 | Auth — invalid PAT | Live | Failure, 401 error | Tests auth error handling |
| 3 | Preflight — sufficient scopes | Live | Success, lists scopes | Token has `repo` + `read:org` |
| 4 | Preflight — missing read:org | Live | Warning/failure | Token missing org access scope |
| 5 | Metadata — list repos in org | Live | Success, returns repos | Fetches `gregmartell-atlan` org repos |
| 6 | Workflow — repos only | Replay | Success, creates Application assets | Extract repos, no wiki/yaml/sbom |
| 7 | Workflow — repos + wiki | Replay | Success, creates Application + ApplicationField | Wiki cloning + page extraction |
| 8 | Workflow — repos + yaml | Replay | Success, extracts YAML files | Searches for .yml/.yaml files |
| 9 | Workflow — full extraction (happy path) | Replay | Success, all asset types created | Repos + wiki + yaml + sbom (if available) |
| 10 | Workflow — empty org short-circuits | Mocked | Success, zero assets | Org has no repos → early exit |
| 11 | Workflow — wiki not initialized | Mocked | Partial success, logs warning | Repo has `has_wiki=true` but wiki.git 404s |
| 12 | Workflow — yaml path 404 | Mocked | Partial success, logs warning | Code search finds file but download 404s |
| 13 | Workflow — rate limit hit (mocked 429) | Mocked | Retry, backoff, eventual success | Tests rate limit handling + retry logic |
| 14 | Replay — pinned workflow history | Replay | Deterministic replay | Uses `workflow_full_extraction.json` fixture |

## Test Types

- **Live**: Calls real GitHub API with credentials from `.env` (only run locally, not in CI unless secrets configured)
- **Replay**: Uses captured HTTP fixtures from `tests/fixtures/github_api/` and replays pre-recorded workflows from `histories/`
- **Mocked**: Uses `pytest-mock` or `httpx-mock` to inject synthetic responses

## Running Tests

### Locally (Live)

```bash
# Set environment variables
export GITHUB_TOKEN=ghp_your_real_token
export GITHUB_ORG=gregmartell-atlan
export ATLAN_API_KEY=your_atlan_key
export ATLAN_BASE_URL=https://your-tenant.atlan.com

# Run live tests (scenarios 1-5)
uv run pytest tests/e2e/test_github_app.py -k live -v
```

### Locally (Replay)

```bash
# Run replay tests (scenarios 6-9, 14)
uv run pytest tests/e2e/test_replay.py -v
```

### CI (Mocked only)

CI runs only mocked tests (scenarios 10-13) to avoid needing real credentials.

```bash
uv run pytest tests/e2e/ -m mocked -v
```

## Captured Fixtures

Fixtures are stored in `tests/fixtures/github_api/`:

- `orgs_repos.json` — Response from `GET /orgs/{org}/repos`
- `repo_readme.json` — Response from `GET /repos/{owner}/{repo}/readme`
- `repo_yaml.json` — Response from `GET /search/code?q=repo:{owner}/{repo}+extension:yml`
- `user.json` — Response from `GET /user` (for auth test)
- `rate_limit.json` — Response from `GET /rate_limit`

Fixtures use REAL data from `gregmartell-atlan` org repos:
- `atlan-usage-analytics`
- `MDLH_Dictionary`
- `MDLH_Dictionary_v1.3`
- `ai-readiness`
- `app-framework_demo`
- `atlan-api-spec`
- `deck-builder`
- `fabric-tags`
- `atlan-global-analytics`

## Workflow Histories

Workflow replay histories are stored in `tests/e2e/histories/`:

- `workflow_full_extraction.json` — Full workflow with all extraction flags enabled (scenario 14)
- `workflow_repos_only.json` — Repos-only extraction (scenario 6)
- `workflow_repos_wiki.json` — Repos + wiki (scenario 7)
- `workflow_repos_yaml.json` — Repos + yaml (scenario 8)

These are generated by running a live workflow once, capturing the event history, and committing it for deterministic replay in CI.

## Adding New Scenarios

1. Run a live test to capture real GitHub API responses
2. Save the responses to `tests/fixtures/github_api/`
3. Save the workflow event history to `tests/e2e/histories/`
4. Add a new replay test case in `test_replay.py`
5. Update this README's scenario matrix

## Notes

- Scenario 4 (missing `read:org` scope) requires creating a token without org access — not automated yet
- SBOM scenarios (9, 14) may fail if the repos don't support GitHub's SBOM API (requires dependency graph enabled)
- Wiki scenarios (7, 9, 11) require repos with initialized wikis — `gregmartell-atlan` repos may not all have wikis
